Appendix -- Essential Git Commands Reference

You will use these commands every single day. Memorize them -- they are your safety net.

Command                What it does
git status             Shows which files you have changed today -- run this first
                       always
git add .              Stages ALL changed files, ready to be committed
git add                Stages only one specific file
apps/web/src/page.tsx
git commit -m "your    Saves a snapshot of all staged changes with a description
message"
git push origin main   Uploads your commits to GitHub -- this is your backup
git pull origin main   Downloads the latest changes from GitHub to your
                       computer
git log --oneline      Shows a compact list of all past commits
git diff               Shows exactly what lines you changed since last commit
git checkout -b        Creates a new branch for experimental work
feature/new-thing
git stash              Temporarily hides your uncommitted changes
git stash pop          Restores the hidden changes

Appendix -- When Things Go Wrong

  Most Common Beginner Problems & Fixes
  'Command not found' for bun/node/git  Close and reopen your terminal after installing
  Environment variables not working  Make sure .env.local is in the apps/web/ folder (not
  root)
  Supabase connection error  Double-check your URL and anon key in .env.local match your
  dashboard
  OpenRouter API 401 error  Your OPENROUTER_API_KEY is wrong or not set -- check
  Vercel environment variables
  git push rejected  Run git pull origin main first to sync, then push again
  Vercel build fails  Check the build log in Vercel dashboard, usually a missing env variable
